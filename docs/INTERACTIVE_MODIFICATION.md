# 交互式代码修改功能

## 📋 概述

现在 Code Agent 支持在代码生成后，继续通过对话来修改代码。系统会自动检测是首次生成还是后续修改，并采用不同的策略。

## ✨ 新功能

### 1. 自动模式检测

系统会自动检测工作模式：

- **首次生成模式**：当 session 中没有现有文件时
  - 从头开始生成代码
  - 创建完整的项目结构

- **修改模式**：当 session 中已有文件时
  - 只修改相关的文件
  - 保留现有代码结构
  - 使用增量更新

### 2. 修改意图分析

在修改模式下，系统会分析用户的修改意图：

- **modify** - 修改现有文件的功能或样式
- **add** - 添加新功能、新文件或新组件
- **delete** - 删除功能或文件
- **refactor** - 重构代码结构
- **fix** - 修复bug或错误

### 3. 智能文件选择

在修改模式下：
- 只修改与用户需求直接相关的文件
- 不会重新生成所有文件
- 保持代码风格一致
- 确保导入/导出兼容

## 🔄 工作流程

### 首次生成流程

```
用户输入 → 意图分析 → 符号检索 → 代码生成 → 验证 → 完成
```

### 后续修改流程

```
用户输入 → 模式检测（发现已有代码）→ 修改意图分析 → 符号检索 → 
增量代码生成（只修改相关文件）→ 验证 → 完成
```

## 📝 使用示例

### 示例 1: 首次生成

**用户输入：**
```
创建一个贪吃蛇游戏
```

**系统行为：**
- 检测到没有现有文件
- 进入首次生成模式
- 生成完整的项目结构（HTML、CSS、JS）

### 示例 2: 修改现有功能

**用户输入：**
```
把蛇的颜色改成红色
```

**系统行为：**
- 检测到已有文件
- 进入修改模式
- 分析意图：modify（修改样式）
- 只修改相关的 CSS 文件
- 保留其他文件不变

### 示例 3: 添加新功能

**用户输入：**
```
添加一个计分系统
```

**系统行为：**
- 检测到已有文件
- 进入修改模式
- 分析意图：add（添加功能）
- 修改 JS 文件添加计分逻辑
- 可能需要修改 HTML 显示分数
- 保留游戏核心逻辑不变

### 示例 4: 修复bug

**用户输入：**
```
修复蛇撞墙后游戏不结束的问题
```

**系统行为：**
- 检测到已有文件
- 进入修改模式
- 分析意图：fix（修复bug）
- 只修改碰撞检测相关的代码
- 保留其他功能不变

## 🎯 技术实现

### 模式检测

```typescript
const currentFiles = await this.getCurrentFiles(sessionId);
const isModificationMode = currentFiles.length > 0;
```

### 修改意图分析

系统使用 LLM 分析用户的修改意图：

```typescript
const modificationIntent = await this.analyzeModificationIntent(prompt, currentFiles);
// 返回: { intent: 'modify'|'add'|'delete'|'refactor'|'fix', targetFiles: [], confidence: 0.8 }
```

### Prompt 优化

在修改模式下，系统会：

1. **提供完整文件内容**（而不是只显示前500字符）
2. **明确指示这是修改模式**
3. **强调只修改相关文件**
4. **要求使用增量更新模式**

```typescript
const modificationModeContext = isModificationMode
  ? `⚠️ 重要：这是修改模式！
- 当前项目已有 ${currentFiles.length} 个文件
- 请只修改与用户需求相关的文件
- 使用增量更新模式，保留现有代码结构
- 确保修改后的代码与现有代码风格一致`
  : '';
```

## 📊 优势

1. **更精确的修改**
   - 只修改相关文件，不会意外破坏其他功能
   - 保持代码结构完整

2. **更好的上下文理解**
   - 提供完整文件内容给 AI
   - AI 能更好地理解现有代码结构

3. **更快的响应**
   - 不需要重新生成所有文件
   - 只处理需要修改的部分

4. **更好的用户体验**
   - 用户可以逐步完善项目
   - 支持迭代式开发

## 🔍 注意事项

1. **文件路径**
   - 确保文件路径正确
   - 使用相对路径（相对于项目根目录）

2. **代码兼容性**
   - 修改后的代码需要与现有代码兼容
   - 确保导入/导出正确

3. **代码风格**
   - 保持与现有代码风格一致
   - 遵循项目的编码规范
